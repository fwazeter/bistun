## I. Strategic Overview

### 1. The "Why"
The `CapabilityManifest` is the authoritative DTO generated by the `LinguisticManager`. It encapsulates resolved linguistic properties, enabling services to execute logic based on language classes (e.g., "Agglutinative") and logical rules rather than hardcoded locale logic.

### 2. System Impact
If this schema is compromised, downstream components receive malformed rendering or processing instructions, causing systemic failures in UI rendering and NLP pipelines.

### 3. Design Patterns
* **Untagged Serialization**: Uses Serde to serialize enums as raw values, ensuring API transparency for non-Rust consumers.
* **Flyweight Reference**: Manifests are constructed from shared, immutable definitions wrapped in `Arc`.

## II. Technical Specification

### 1. Primary Interface: `CapabilityManifest`
The root structure provides the resolved identity, functional traits, and operational rules.

| Field | Type | Purpose |
| :--- | :--- | :--- |
| `resolved_locale` | `String` | The BCP 47 tag resulting from taxonomic resolution. |
| `traits` | `Map<TraitKey, TraitValue>` | The dictionary of "Linguistic DNA" traits. |
| `rules` | `Map<String, LmsRule>` | **[NEW]** Logical directives for algorithmic execution (Ref: [013-LMS-RULE]). |
| `resources` | `Map<String, String>` | **[NEW]** Mappings of Logical Resource IDs to Physical Paths (Ref: [014-LMS-BRDG]). |
| `metadata` | `Map<String, String>` | Observability data (registry version, latency). |

### 2. Golden Set & Standard Enums
The "Golden Set" ensures a predictable contract for all consumers.

| Trait Key | Value Type | Description |
| :--- | :--- | :--- |
| `PRIMARY_DIRECTION` | `Direction` | UI directionality (LTR, RTL, BIDI). |
| `SEGMENTATION_STRATEGY` | `SegType` | Boundary detection (SPACE, DICTIONARY). |
| `MORPHOLOGY_TYPE` | `MorphType` | Linguistic structure (AGGLUTINATIVE, TEMPLATIC). |
| `NORMALIZATION_DEFAULT`| `NormType` | Default Unicode normalization form (NFC, NFD). |

## III. Execution & Integrity

### 1. Performance Budget
* **Target Latency**: < 1ms p99 resolution.
* **Complexity**: O(1) retrieval from the hydrated manifest.

### 2. Enforcement & Validation (LMS-VAL)
* **Level C (Runtime)**: Verifies that the `traits` map contains mandatory rendering keys (Direction, Segmentation) before returning to the caller.
* **Rule Integrity**: Rules in the `rules` map must be logically compatible with the `MORPHOLOGY_TYPE`.

### 3. Observability (LMS-OPS)
* **Telemetry Keys**: `resolution_time_ms`, `registry_version`, `resolution_path`.
