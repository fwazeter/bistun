## I. Overview

The Bistun LMS project is designed for a **Hybrid Authoring** model where humans and AI coding agents (e.g., Cursor, Copilot, or specialized LLMs) co-create the "System of Record". To prevent architectural drift, this standard defines the mandatory constraints and interaction patterns that any AI agent must follow when proposing or editing code in this repository.

**Location**: `docs/standards/LMS-AI.md`

---

## II. Mandatory Reasoning Constraints

Any AI agent interacting with the `src/` tree must prioritize these three logic gates above all else:

### 1. The "System of Record" Rule
Every change must be narrated as a permanent update to a linguistic truth. AI agents must never suggest "quick hacks" or "placeholder logic" that bypasses the **LMS-DOC** or **LMS-TEST** requirements.

### 2. High-Water Mark Alignment
When modifying aggregation logic, the agent must check for **Complexity Ranking** violations.
* **Constraint**: An AI must ensure that `SEGMENTATION_STRATEGY` always defaults to the highest rank found in the script pool.

### 3. Performance Budget Awareness
Any logic that introduces $O(N^2)$ complexity or heavy heap allocations in the resolution path is a **System Failure**.
* **Constraint**: AI agents must verify that proposed changes do not breach the **$<1\text{ms}$** p99 resolution target.

---

## III. Interaction Protocols

### 1. The "Logic Trace" First Pattern
Before providing code, the AI agent must output its internal reasoning steps in a format that maps directly to the `# Logic Trace` required by **LMS-DOC**.

### 2. Standard Scaffolding
AI agents must use `just new-module <path>` to bootstrap files, ensuring the `TEMPLATE.rs` standard is enforced from the first byte.

---

## IV. AI-Agent System Prompt (Contextual Baseline)

When initializing a new session with an AI agent, the following context should be injected:

```text
You are an expert Rust engineer co-authoring the Bistun LMS, a Linguistic System of Record. 
Your instructions:
1. Reference Blueprints [001-012] for every architectural decision.
2. Adhere to LMS-DOC (Narrative Code) and LMS-TEST (Isolation/Mocking) standards.
3. Maintain the <1ms performance budget for all capability resolution logic.
4. Use the High-Water Mark Strategy for multi-script trait aggregation.
5. All code must be 'Novice-Friendly' through step-by-step logic tracing.
```

---

## V. Quality Assurance Gate (AI Alignment)

* **Refusal Criteria**: An AI agent should refuse to implement a feature if the prompt contradicts the **ROADMAP** priority order or the **Truth Hierarchy**.
* **Review Verification**: PRs generated by AI must be verified against this document to ensure no "Hallucinated Capabilities" were added that deviate from the **Linguistic DNA** model.
